MiniMax API 新手购买与接入教程
核对时间：2026-03-18

这份文件是给第一次接触 MiniMax 的人准备的。
目标不是讲很多概念，而是让你能把 ExamNova 跑通。

一、先看结论

如果你是为了用 ExamNova，这条路线最稳：

1. 注册 / 登录 MiniMax 开发平台
2. 看官方价格页，决定先试用还是先充值
3. 创建 API Key
4. 回到 ExamNova 设置页
5. 填 Base URL、Model、API Key
6. 点“测试连接”

对这个项目来说，最重要的是“拿到可用的 API Key”，不是研究所有套餐。


二、官方入口

1. Quickstart
https://platform.minimaxi.com/docs/guides/quickstart

2. 定价页
https://www.minimaxi.com/pricing

3. OpenAI 兼容接口文档
https://platform.minimaxi.com/docs/api-reference/text-openai-api

4. API FAQ
https://platform.minimaxi.com/docs/api-reference/basic-information/faq


三、到底是“买 API”还是“充值账号”

更准确地说，通常不是单独买一个 API 文件，而是：

1. 你先注册 MiniMax 平台账号
2. 再看官方价格页和套餐
3. 决定是否充值余额或购买适合你的套餐
4. 然后在平台里创建 API Key
5. 最后用这个 Key 调接口

所以你真正要做的是：

- 先有账号
- 再有可用额度
- 再创建 Key

如果没有额度，API FAQ 里提到会遇到余额不足之类的报错。


四、第一次购买 / 充值，怎么选

如果你只是为了把 ExamNova 跑起来，建议这样理解：

1. 先看官方价格页，不要死记第三方截图
2. 如果平台给了试用额度，可以先用试用额度跑通
3. 如果你要稳定长期使用，再充值或选适合自己的套餐

对 ExamNova 当前版本来说，最核心的是文本模型调用。
也就是说，你不用一上来就追求最贵、最全的方案。

更实用的原则是：

- 先能连通
- 先能稳定出题解
- 后面再根据花费和速度换模型


五、创建 API Key

拿到额度后，再去平台里创建 API Key。

建议你注意这几件事：

1. Key 只放在你自己的设备里
2. 不要截图发群
3. 不要写进 GitHub
4. 如果怀疑泄露，立刻删除并重建

ExamNova 现在不会把你的运行数据写到系统默认程序目录。
当前版本会把数据写在：

开发模式：
<repo>/.examnova-data

桌面版：
<ExamNova.exe 所在目录>/data


六、在 ExamNova 里怎么填

打开设置页后，建议这样填：

本地后端地址：
http://127.0.0.1:8000

LLM Base URL：
https://api.minimaxi.com/v1

Model：
MiniMax-M2.5

API Key：
你刚创建的 MiniMax Key

说明：

1. 这个项目当前默认值写的是 MiniMax-M2.5，是为了先求稳
2. 如果你后面要换官方当前支持的其他文本模型，也可以改
3. 如果你只是想先看页面，保留演示模式即可，不必先充值


七、为什么这里写 OpenAI 兼容接口

因为 ExamNova 当前接的是 OpenAI 风格接口。
MiniMax 官方也提供了 OpenAI 兼容文档。

这意味着你在项目里主要关心这三项：

- Base URL
- Model
- API Key

对新手来说，这比直接看一大堆底层接口文档更容易上手。


八、常见问题

1. “连接失败：Failed to fetch”
先检查本地后端是否真的启动了，不一定是 MiniMax 本身有问题。

2. “余额不足 / 额度不足”
优先回 MiniMax 平台确认账户余额、套餐状态和当前 Key 是否可用。

3. “API Key 明明填了还是不行”
检查有没有多复制空格、Key 是否失效、Base URL 是否填错。

4. “能不能先不买”
可以。先开演示模式看界面和流程，再决定是否充值。


九、给开源使用者的建议

如果你准备把这个项目发到 GitHub，建议把这句话写清楚：

“本项目不内置任何第三方模型密钥，使用者需自行注册 MiniMax 平台并配置自己的 API Key。”

这样别人一看就知道：

1. 你没有把密钥写死
2. 仓库是安全的
3. 接入方式是清楚的


十、最短操作版

如果你只想看最短步骤，就照这个做：

1. 打开 Quickstart 页面
2. 注册并登录
3. 打开定价页，决定是否先充值
4. 创建 API Key
5. 回到 ExamNova 设置页
6. 填：
   Base URL = https://api.minimaxi.com/v1
   Model = MiniMax-M2.5
   API Key = 你的 Key
7. 点“测试连接”

如果你能走到这一步，就已经够把项目跑起来了。
